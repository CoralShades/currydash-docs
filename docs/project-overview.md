# CurryDash Documentation - Project Overview

## About This Project

CurryDash Documentation is the central documentation hub for the CurryDash food delivery platform. It aggregates and organizes documentation from multiple source repositories, providing a unified reference for developers, vendors, and administrators.

## Architecture Overview

This is a **component-based documentation platform** using:

- **Next.js App Router** for routing and rendering
- **Nextra Theme** for documentation UI/UX
- **MDX Content** for documentation pages
- **Headless API** for programmatic content access
- **Pagefind** for client-side full-text search

## Target Audiences

The documentation serves three primary audiences:

### Developers
Technical documentation for building and integrating with CurryDash:
- Backend architecture (Laravel)
- Frontend development (Flutter)
- API reference and integration guides
- Deployment and DevOps

### Vendors
Guides for restaurants and sellers using the platform:
- Seller portal usage
- Menu management
- Order processing
- Payment and analytics

### Administrators
Documentation for platform operators:
- User management
- Vendor oversight
- Platform configuration
- Analytics and reporting

## Key Features

### Cross-Repository Sync
Documentation automatically receives updates when code changes are committed to source repositories via GitHub Actions `repository_dispatch`.

### BMAD Method Integration
Built-in support for BMAD workflows:
- `*paige di` - Initialize documentation
- `*paige da` - Analyze stories for doc needs
- `*paige ds` - Sync pending changes

### Semantic Versioning
Automated versioning and changelog generation using Conventional Commits and Semantic Release.

### Full-Text Search
Client-side search powered by Pagefind, a Rust-based static search solution.

## Getting Started

```bash
# Clone repository
git clone https://github.com/CoralShades/currydash-docs.git
cd currydash-docs/docs

# Install dependencies
npm install

# Start development server
npm run dev
```

Visit **http://localhost:3000** to view the documentation.
