## Platform Repositories

CurryDash is organized across multiple repositories:

| Repository | Purpose | Technologies |
|------------|---------|--------------|
| [CurryDash-Admin-Seller-Portal](https://github.com/CoralShades/CurryDash-Admin-Seller-Portal) | Backend API & Admin Portal | Laravel 9.x, MySQL, Redis |
| [CurryDash-UserApp-Mobile_Web-Flutter](https://github.com/CoralShades/CurryDash-UserApp-Mobile_Web-Flutter) | Customer Mobile & Web Apps | Flutter, Dart |
| [currydash-docs](https://github.com/CoralShades/currydash-docs) | Platform Documentation | Nextra 4, Next.js |

## Quick Start by Role

### For Backend Developers

```bash
git clone https://github.com/CoralShades/CurryDash-Admin-Seller-Portal.git
cd CurryDash-Admin-Seller-Portal/backend
composer install && php artisan migrate
```

[Backend Setup Guide](/developers/backend)

### For Frontend Developers

```bash
git clone https://github.com/CoralShades/CurryDash-UserApp-Mobile_Web-Flutter.git
cd CurryDash-UserApp-Mobile_Web-Flutter
flutter pub get && flutter run
```

[Flutter Setup Guide](/developers/frontend)

### For Vendors

Access the Seller Portal at **https://seller.currydash.com**

[Vendor Onboarding Guide](/vendors/getting-started)

### For Admins

Access the Admin Portal at **https://admin.currydash.com**

[Admin Guide](/admins/getting-started)

## Platform Architecture

```
+----------------------------------------------------------+
|  Flutter Mobile/Web Apps                                  |
|  Repository: CurryDash-UserApp-Mobile_Web-Flutter         |
+-----------------------------+----------------------------+
                              | REST API
+-----------------------------v----------------------------+
|  Laravel Backend API + Admin Portal                      |
|  Repository: CurryDash-Admin-Seller-Portal               |
+----------------------------------------------------------+
|  Cloud SQL (MySQL) | Memorystore (Redis)                 |
|  Google Cloud Platform                                   |
+----------------------------------------------------------+
```

## Documentation Updates

This documentation is automatically synchronized with both code repositories:
- Commits to `CurryDash-Admin-Seller-Portal` update Backend/Admin/Vendor docs
- Commits to `CurryDash-UserApp-Mobile_Web-Flutter` update Frontend docs

Using BMAD Method with semantic-release for versioning.
