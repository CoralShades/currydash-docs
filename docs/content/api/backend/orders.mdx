# Orders

Comprehensive order management endpoints for placing orders, real-time tracking, viewing order history, and retrieving detailed order information.

<Callout type="info">
  All order endpoints require authentication via JWT token (`auth:api` guard) unless specified otherwise.
</Callout>

## Overview

The CurryDash order management system provides:

- **Order Placement**: Create orders from cart with payment processing
- **Real-Time Tracking**: Track order status and delivery person location
- **Order History**: View past orders with filtering and pagination
- **Order Details**: Retrieve complete order information including items and delivery details
- **Schedule Orders**: Support for scheduled delivery times
- **Multiple Payment Methods**: Cash, Stripe, PayPal, and Wallet payments

### Order Status Flow

| Status | Description | Next Status |
|--------|-------------|-------------|
| `pending` | Order placed, awaiting restaurant confirmation | `confirmed` or `canceled` |
| `confirmed` | Restaurant confirmed, preparing food | `processing` |
| `processing` | Food being prepared | `out_for_delivery` |
| `out_for_delivery` | Assigned to delivery person | `delivered` |
| `delivered` | Order completed successfully | - |
| `canceled` | Order canceled by customer/restaurant | - |
| `failed` | Payment or delivery failed | - |

### Payment Status

| Status | Description |
|--------|-------------|
| `paid` | Payment successfully processed |
| `unpaid` | Cash on delivery (payment pending) |
| `failed` | Payment processing failed |
| `refunded` | Payment refunded after cancellation |

## Endpoints

### Place Order

Create a new order from the user's cart with payment processing and delivery details.

#### POST `/api/v1/orders`

**Controller**: `App\Http\Controllers\Api\V1\OrderController@place_order`

**Authentication**: Required (`auth:api`)

**Rate Limit**: 10 requests/minute per user

**Request Body**:

```json
{
  "payment_method": "stripe",
  "order_amount": 143.55,
  "order_type": "delivery",
  "delivery_address_id": 12,
  "distance": 2.3,
  "schedule_at": null,
  "order_note": "Please ring the doorbell",
  "coupon_discount_amount": 10.00,
  "coupon_code": "WELCOME10"
}
```

**Code Examples**:

<Tabs items={['PHP/Laravel', 'cURL']}>
  <Tabs.Tab>
    ```php
    use Illuminate\Support\Facades\Http;
    use Illuminate\Http\Client\RequestException;

    try {
        $accessToken = session('access_token');

        if (!$accessToken) {
            return response()->json([
                'success' => false,
                'message' => 'Not authenticated',
            ], 401);
        }

        $response = Http::withToken($accessToken)
            ->post('https://api.currydash.au/api/v1/orders', [
                'payment_method' => 'stripe',
                'order_amount' => 143.55,
                'order_type' => 'delivery',
                'delivery_address_id' => 12,
                'distance' => 2.3,
                'schedule_at' => null,
                'order_note' => 'Please ring the doorbell',
                'coupon_discount_amount' => 10.00,
                'coupon_code' => 'WELCOME10',
            ]);

        if ($response->successful()) {
            $data = $response->json();

            return response()->json([
                'success' => true,
                'order_id' => $data['order_id'],
                'order_status' => $data['order_status'],
                'payment_status' => $data['payment_status'],
                'message' => $data['message'],
            ], 201);
        }

        // Handle empty cart (400)
        if ($response->status() === 400) {
            $errors = $response->json('errors', []);
            $errorCode = $errors[0]['code'] ?? 'order-error';

            return response()->json([
                'success' => false,
                'message' => $errors[0]['message'] ?? 'Unable to place order',
                'code' => $errorCode,
            ], 400);
        }

        // Handle payment failed (402)
        if ($response->status() === 402) {
            $errors = $response->json('errors', []);
            return response()->json([
                'success' => false,
                'message' => 'Payment processing failed. Please check your payment method.',
                'errors' => $errors,
            ], 402);
        }

        // Handle zone/address issues (404)
        if ($response->status() === 404) {
            $errors = $response->json('errors', []);
            return response()->json([
                'success' => false,
                'message' => $errors[0]['message'] ?? 'Service not available',
                'errors' => $errors,
            ], 404);
        }

        // Handle validation errors (422)
        if ($response->status() === 422) {
            $errors = $response->json('errors', []);
            return response()->json([
                'success' => false,
                'message' => 'Validation failed',
                'errors' => $errors,
            ], 422);
        }

        // Handle other errors
        return response()->json([
            'success' => false,
            'message' => 'Failed to place order',
        ], $response->status());

    } catch (RequestException $e) {
        // Handle network or timeout errors
        Log::error('Place order request failed', [
            'error' => $e->getMessage(),
        ]);

        return response()->json([
            'success' => false,
            'message' => 'Unable to connect to order service',
        ], 500);
    }
    ```

**Field Descriptions**:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `payment_method` | string | Yes | Payment method: `cash_on_delivery`, `stripe`, `paypal`, `wallet` |
| `order_amount` | decimal | Yes | Total order amount including all charges |
| `order_type` | string | Yes | Order type: `delivery` or `takeaway` |
| `delivery_address_id` | integer | Conditional | Required if `order_type` is `delivery` |
| `distance` | decimal | No | Distance in km (calculated automatically if not provided) |
| `schedule_at` | datetime | No | Scheduled delivery time (ISO 8601 format) |
| `order_note` | string | No | Special instructions (max 500 characters) |
| `coupon_discount_amount` | decimal | No | Discount amount from applied coupon |
| `coupon_code` | string | No | Coupon code applied to order |

**Validation Rules**:

```php
[
    'payment_method' => 'required|in:cash_on_delivery,stripe,paypal,wallet',
    'order_amount' => 'required|numeric|min:0',
    'order_type' => 'required|in:delivery,takeaway',
    'delivery_address_id' => 'required_if:order_type,delivery|exists:addresses,id',
    'distance' => 'nullable|numeric',
    'schedule_at' => 'nullable|date|after:now',
    'order_note' => 'nullable|string|max:500',
    'coupon_discount_amount' => 'nullable|numeric|min:0',
    'coupon_code' => 'nullable|exists:coupons,code',
]
```

**Business Logic**:

<Steps>

### Validate Cart
Cart must not be empty and all items must be from the same restaurant.

### Validate Delivery Address
For delivery orders, verify address is within restaurant's service zone.

### Check Minimum Order
Verify order meets restaurant's minimum order amount.

### Apply Coupon
Validate coupon code and apply discount if valid.

### Calculate Delivery Charge
Calculate delivery fee based on distance and zone.

### Process Payment
Process payment via selected method (Stripe, PayPal, or Wallet).

### Create Order
Create order record with all items and details.

### Clear Cart
Remove all items from user's cart.

### Send Notifications
Trigger notifications to customer, vendor, and available delivery personnel.

</Steps>

### Track Order

Track order status in real-time including delivery person location and estimated delivery time.

#### GET `/api/v1/orders/track`

**Controller**: `App\Http\Controllers\Api\V1\OrderController@track`

**Authentication**: Required (`auth:api`)

**Rate Limit**: 60 requests/minute per user

**Query Parameters**:

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `order_id` | integer | Yes | The order ID to track |

**Example Request**:

```
GET /api/v1/orders/track?order_id=789
```

**Code Examples**:

<Tabs items={['PHP/Laravel', 'cURL']}>
  <Tabs.Tab>
    ```php
    use Illuminate\Support\Facades\Http;
    use Illuminate\Http\Client\RequestException;

    try {
        $accessToken = session('access_token');

        if (!$accessToken) {
            return response()->json([
                'success' => false,
                'message' => 'Not authenticated',
            ], 401);
        }

        $orderId = 789; // Get from request

        $response = Http::withToken($accessToken)
            ->get('https://api.currydash.au/api/v1/orders/track', [
                'order_id' => $orderId,
            ]);

        if ($response->successful()) {
            $data = $response->json();

            return response()->json([
                'success' => true,
                'data' => $data,
            ]);
        }

        // Handle order not found (404)
        if ($response->status() === 404) {
            $errors = $response->json('errors', []);
            return response()->json([
                'success' => false,
                'message' => 'Order not found or does not belong to you',
                'errors' => $errors,
            ], 404);
        }

        // Handle unauthorized (401)
        if ($response->status() === 401) {
            return response()->json([
                'success' => false,
                'message' => 'Invalid or expired token',
            ], 401);
        }

        // Handle other errors
        return response()->json([
            'success' => false,
            'message' => 'Failed to track order',
        ], $response->status());

    } catch (RequestException $e) {
        // Handle network or timeout errors
        Log::error('Track order request failed', [
            'order_id' => $orderId ?? null,
            'error' => $e->getMessage(),
        ]);

        return response()->json([
            'success' => false,
            'message' => 'Unable to connect to order service',
        ], 500);
    }
    ```

### Order History

Retrieve paginated list of user's past orders with filtering options.

#### GET `/api/v1/orders/history`

**Controller**: `App\Http\Controllers\Api\V1\OrderController@get_order_list`

**Authentication**: Required (`auth:api`)

**Rate Limit**: 60 requests/minute per user

**Query Parameters**:

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `limit` | integer | No | Number of orders per page (default: 10, max: 50) |
| `offset` | integer | No | Pagination offset (default: 0) |
| `status` | string | No | Filter by order status |
| `order_type` | string | No | Filter by order type: `delivery` or `takeaway` |

**Example Request**:

```
GET /api/v1/orders/history?limit=10&offset=0&status=delivered
```

**Path Parameters**:

|-----------|------|----------|-------------|
